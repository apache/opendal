18//! The core concepts of OpenDAL's public API.
19//!
20//! OpenDAL provides a unified abstraction that helps developers access all storage services.
21//!
22//! There are two core concepts in OpenDAL:
23//!
24//! - [`Builder`]: Builder accepts a series of parameters to set up an instance of underlying services.
25//!   You can adjust the behaviour of underlying services with these parameters.
26//! - [`Operator`]: Developer can access underlying storage services with manipulating one Operator.
27//!   The Operator is a delegate for underlying implementation detail, and provides one unified access interface,
28//!   including `read`, `write`, `list` and so on.
29//!
30//! If you are interested in internal implementation details, please have a look at [`internals`][super::internals].
31//!
32//! # Builder
33//!
34//! Let's start with [`Builder`].
35//!
36//! A `Builder` is a trait that is implemented by the underlying services. We can use a `Builder` to configure and create a service.
37//! Developer can only create one service via Builder, in other words, Builder is the only public API provided by services.
38//! And other detailed implementation will be hidden.
39//!
40//! ```text
41//! ┌───────────┐                 ┌───────────┐
42//! │           │     build()     │           │
43//! │  Builder  ├────────────────►│  Service  │
44//! │           │                 │           │
45//! └───────────┘                 └───────────┘
46//! ```
47//!
48//! All [`Builder`] provided by OpenDAL is under [`services`][crate::services], we can refer to them like `opendal::services::Memory`.
49//! By right the builder will be named like `OneServiceBuilder`, but usually we will export it to public with renaming it as one
50//! general name. For example, we will rename `S3Builder` to `S3` and developer will use `S3` finally.
51//!
52//! For example:
53//!
54//! ```no_run
55//! use opendal_core::services::Memory;
56//!
57//! let builder = Memory::default();
58//! ```
59//!
60//! # Operator
61//! The [`Operator`] is a delegate for Service, the underlying implementation detail that implements [`Access`][crate::raw::Access],
62//! and it also provides one unified access interface.
63//! It will hold one reference of Service with its all generic types erased by OpenDAL,
64//! which is the reason why we say the Operator is the delegate of one Service.
65//!
66//! ```text
67//!                   ┌────────────────────┐
68//!                   │      Operator      │
69//!                   │         │delegate  │
70//! ┌─────────┐ build │         ▼          │ rely on ┌─────────────────────┐
71//! │ Builder ├───────┼──►┌────────────┐   │◄────────┤ business logic code │
72//! └─────────┘       │   │  Service   │   │         └─────────────────────┘
73//!                   └───┴────────────┴───┘
74//! ```
75//!
76//! `Operator` can be built from `Builder`:
77//!
78//! ```no_run
79//! # use opendal_core::Result;
80//! use opendal_core::services::Memory;
81//! use opendal_core::Operator;
82//!
83//! # fn test() -> Result<()> {
84//! let builder = Memory::default();
85//!
86//! let op = Operator::new(builder)?.finish();
87//! # Ok(())
88//! # }
89//! ```
90//!
91//! - `Operator` has it's internal `Arc`, so it's **cheap** to clone it.
92//! - `Operator` doesn't have generic parameters or lifetimes, so it's **easy** to use it everywhere.
93//! - `Operator` implements `Send` and `Sync`, so it's **safe** to send it between threads.
94//!
95//! After get an `Operator`, we can do operations on different paths.
96//!
97//!
98//! ```text
99//!                            ┌──────────────┐
100//!                  ┌────────►│ read("abc")  │
101//!                  │         └──────────────┘
102//! ┌───────────┐    │
103//! │ Operator  │    │         ┌──────────────┐
104//! │ ┌───────┐ ├────┼────────►│ write("def") │
105//! │ │Service│ │    │         └──────────────┘
106//! └─┴───────┴─┘    │
107//!                  │         ┌──────────────┐
108//!                  └────────►│ list("ghi/") │
109//!                            └──────────────┘
110//! ```
111//!
112//! We can read data with given path in this way:
113//!
114//! ```no_run
115//! # use opendal_core::Result;
116//! use opendal_core::services::Memory;
117//! use opendal_core::Operator;
118//!
119//! # async fn test() -> Result<()> {
120//! let builder = Memory::default();
121//!
122//! let op = Operator::new(builder)?.finish();
123//! let bs: Vec<u8> = op.read("abc").await?;
124//! # Ok(())
125//! # }
126//! ```
127//!
128//! [`Builder`]: crate::Builder
129//! [`Operator`]: crate::Operator
